# Business Case

## Expected Outcome (Maximum of 5 slides):

1. A solution architecture for a production level application using cloud resources (e.g. Azure, AWS or GCP) or
resource function (e.g. Storage, API, Event Broker).
2. An overview of the design flow, key decisions and the approach for solution evaluation (Offline & Online).
3. A Jupyter notebook or frontend prototype for real-time interaction (We will run this on a call with you)

## Problem Statement:
Build a tool that aids with the extraction of insights from financial documents (contains text, tables and images).
The platform must:
1. Engage in conversations within and outside of the knowledge base. For knowledge base queries, cite relevant
documents pages and relevance. No citations required for information retrieved outside knowledge base.
2. Provide response based on user intended document segment or seek clarification if unclear.

## Scope
1. Use the financial document here (https://www.bmo.com/ir/archive/en/bmo_ar2025_MDA.pdf)
2. Segment Information based on document headings
3. Create a vector database with appropriate tags for streamlined information retrieval (Headings, Table vs Text,
etc)
